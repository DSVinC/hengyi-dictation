# 恒一听写系统 - 设计文档

## 仓库结构
```
hengyi-dictation/
├── index.html
├── css/style.css
├── js/app.js
└── data/
    ├── chinese/
    │   ├── lessons.json
    │   └── L01.json
    └── english/
        ├── units.json
        └── U01.json
```

## 页面设计（3页）
1. **听写模式**：选科目 → 选课/单元 → 勾选词 → 生成清单
2. **词库管理**：查看各课/单元词语和进度
3. **进度总览**：掌握率统计、高频错词

## 数据结构
- 轮次：0(新) → 1(1天) → 2(3天) → 3(7天) → 4(15天) → 5(30天) → ✅(掌握)
- 错：回轮次1，对：轮次+1

## 听写清单生成规则

### 词数上限
- **每日上限：20 词**（避免负担过重）

### 优先级规则
1. **R1 到期词优先**：必须听写，可突破上限
2. **手动勾选新词**：纳入剩余名额
3. **手动勾选复习词**：优先纳入剩余名额
4. **R2-R5 到期词**：按轮次排序纳入剩余名额

### 勾选页面结构
- **新词区块**：当前课/单元的 R0 词，可自由勾选
- **到期复习-R1 区块**：跨课/单元的 R1 词，默认勾选且 disabled（不可取消）
- **到期复习-R2+ 区块**：跨课/单元的 R2+ 词，可勾选

### 延期处理
- 超出上限的词延期到明天
- 显示延期提示：⏸️ 延期词语: X 个（明天再复习）

## 数据流
仓库 JSON → GitHub Pages 读取 → 网页展示 → 灵爪更新 → 仓库提交
